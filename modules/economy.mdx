This page provides the essential configuration options for Ducky's **Server Economy** module, which allows your members to participate in a fun, engaging in-server economy. The module creates a centralized system for collecting and managing **:quack: Quacks**, ensuring all transactions and balances are properly tracked. To enable and configure this module, use the **`/setup`** command and navigate to the **Server Economy** page.

### ER:LC-Integrated Bounties

**ER:LC-Integrated bounties** allows your server members to set bounties on in-game players, and rewarding the player who kills them with Economy currency. Multiple players can add a bounty on a player via the `/bounty add` command. Active bounties can be viewed via the **`/bounty list <player>`**. Server managers can remove + refund a bounty placed on a player.

<Note>
  The bounty claimer must be linked via Ducky to receive the reward. Your server must have the Economy module enabled, and your [ER:LC private server API Key](/tutorials/linkerlc) must be linked.
</Note>

### Commands

Ducky has a vast ammount of economy ommands, these include:

`/deposit` (`d!dep`): _Deposit quacks into your bank!_

`/withdraw` (`d!wd`, `d!with`): _Withdraw quacks from your bank!_

`/shop` (`d!store`): _Open the shop._

`/donate` (`d!give`, `d!transfer`): _Donate quacks to the given user!_

`/balance` (`d!bal`, `d!quacks`): _Show how many quacks you have._

`/work`: _Work to make some quacks!_

`/beg`: _Beg for quacks._

`/leaderboard` (`d!lb`): _Show your server's economy leaderboard!_

`/slotmachine` (`d!slots`): _Gamble with your quacks in a slot machine._

`/buy` (`d!purchase`): _Buy an item from the shop._

`/inventory` (`d!inv`): _View your inventory._

`/rob`: _Robs the given user!_

`/crime`: _Commits a crime!_

`/use`: _Use an item in your inventory._

`/daily`: _Claim your daily bonus!_

`/blackjack` (`d!bj`, `d!bjack`): _Gamble with your quacks in a game of Blackjack._

### Currency

**Currency** is the emoji and/or text which appears before the amount of Ducky money the user holds. For instance, changing your currency to: "üê£" will show all amounts of money as "üê£ 100".

### Initial Amount

The **initial amount** is the amount of money a user starts with. By default, this is set to 0, but setting it to 100 will grant all new users 100 currency.

### Amounts

**Amounts** is the range or static number of currency a user can get for various actions. You can currently edit **work**, **crime**, **beg**, **daily**.

### Custom Cooldowns

**Custom Cooldowns** are used to set a Cooldown on the commands so spamming won't occur.

### Collection Roles

### Role

The **role** is the role which can collect the specified amount.

### Amount

The **amount** is the amount which can be collected each time.

### Target

The **target** is where the currency collected will go. **Wallet** will send the currency to the user's wallet (undeposited). **Bank** will send the currency to the user's bank (deposited).

### Cooldown

The **cooldown** is the amount of time in between when a user can collect their currency.

### Economy Log Channel

The **economy log channel** is the channel which all logs of economy purchases and item uses are sent to.

### Blacklisted Roles

**Blacklisted roles** are roles which cannot partake in the server economy whatsoever.

### Custom Replies

The **Custom Replies** feature allows you to seamlessly create custom replies for the Work, Beg, Rob and Crime commands. You can create any reply, just rember to insert these variables: `{ammount}` and `{target}`. `{target}` can only be used for rob command, `{ammount}`must be used in all custom replies.

### Shop

The **shop** feature allows you to create custom shop items for your users and gives the ability to purchase these items for currency in exchange for roles.

### Name

The **name** of a shop item is what will show in the shop, inventory, and anywhere else you see shop items. The name should be something easily identifiable.

### Description

The **description** of a shop item is what users see in the shop and inventory. The description should describe what the item does and what perks it gives.

### Price

The **price** of a shop item is the amount of currency it costs to purchase that item.

### Role

The **role** of a shop item is the Discord role given to the user when the item is used. This role can be functional or purely cosmetic. The role is only given to the user once they use the item.

### Use on Purchase

**Use on purchase** determines when the user recieves the item's role. When toggled on, the user will immediately receive the role of the shop item when purchasing it. When toggled off, the user must go into their inventory and use the item.

### Message

The **message** is the message which is sent when a user purchases an item. The message is text-only.

### ID

The \*\*ID \*\*is a unique identifer for the shop ID across all Ducky servers. There is no other shop item in any Ducky server that has the same ID.

### Multipliers

### Role

The **role** is the role which is affected by the multiplier. This includes role granted by other shop items.

### Multiplier

The **multiplier** is the amount which all currency of the user affected is multiplied by.

### Reset Economy

**Reset economy** will wipe all economy congiuration, including shop items, multipliers, cooldowns, responses, and every other configuration mentioned on this page.

<Danger>
  Reset economy is a dangerous and destructive option. Using reset economy is irreversible; you cannot recover your configuration after it is reset.
</Danger>
