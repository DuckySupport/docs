### Base URL

`api.duckybot.xyz`

### Ratelimits

The base ratelimit is 20 requests per 10 seconds. Do not hardcode any values. Use the following headers found in every response:

<ResponseField name="header" type="x-ratelimit-limit">
  The amount of requests that can be made before receiving a 429.
</ResponseField>

<ResponseField name="header" type="x-ratelimit-remaining">
  The number of remaining requests that can be made.
</ResponseField>

<ResponseField name="header" type="x-ratelimit-reset">
  Epoch timestamp when the rate limit resets.
</ResponseField>

### 429 response:
```json
{
  "code": 429,
  "message": "Too many requests, please try again later."
}
```

### `GET /users/:id`

<Callout icon="pencil">
  Returns information about a Discord or Roblox user.
</Callout>

<ResponseField name="param" type="id" required>
  A Discord user ID, Roblox user ID, or @me. If you use @me, the token header is required.
</ResponseField>

<ResponseField name="header" type="Token">
  A Ducky session token.
</ResponseField>

### `DELETE /links`

<Callout icon="pencil">
  Delete a link. You must provide either a session token or a roblox-code.
</Callout>

<ResponseField name="header" type="token">
  The session token of the user.
</ResponseField>

<ResponseField name="header" type="roblox-code">
  The Roblox OAuth2 access code of the user.
</ResponseField>

```json expandable lines wrap
{
  "code": 200,
  "message": "Roblox user has been successfully unlinked."
}
```

### `POST /auth`

<Callout icon="pencil">
  Generate a Ducky session token.
</Callout>

<ResponseField name="header" type="discord-code" required>
  The Discord OAuth2 code of the user.
</ResponseField>

```json expandable lines wrap
{
    "code": 200,
    "data": {
        "user": {
            "id": "782235114858872854",
            "username": "bobbibones",

```

### `ANY /guilds/:id/:action/:subaction`

<Callout icon="pencil">
  The format of all `/guilds` requests.
</Callout>

<ResponseField name="param" type="id">
  The ID of the target guild.
</ResponseField>

<ResponseField name="param" type="action">
  The base feature you want to interact with.
</ResponseField>

<ResponseField name="param" type="subaction">
  Specifies what to do regarding that feature.
</ResponseField>

<ResponseField name="header" type="token" required>
  The session token of the user.
</ResponseField>
